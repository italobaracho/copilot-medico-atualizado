O Objetivo Slide 2/7
🌪️

O Caos de Antes

Subir arquivos direto em branches principais, commits genéricos sem contexto e conflitos de mesclagem constantes que travam o desenvolvimento.

🛡️

Fluxo Seguro

Criação de branches de desenvolvimento e testes separadas para blindar o ambiente de produção contra bugs.

⚡

Agilidade

Padronizar commits permite automatizar relatórios de alterações e ajuda quem for ler o código a entender na hora o que mudou.

Copilot Médico Dica: commits bem feitos evitam dores de cabeça
Estrutura de Branches Slide 3/7
🛠️

1. dev

Nosso ponto de encontro diário. Toda funcionalidade nova criada pela equipe é integrada e testada inicialmente aqui.

🧪

2. homolog

Nosso palco de testes finais. Simula o ambiente real para validações e controle de qualidade (QA) antes do lançamento.

💎

3. main

Código sagrado de produção. Contém apenas versões estáveis de lançamentos aprovadas e carimbadas.

Passo a Passo Slide 4/7

Como começar uma tarefa (Feature) no terminal:

Terminal - Git Bash
1 # Garanta que está na branch de desenvolvimento
2 git checkout dev
3 # Baixe as alterações mais recentes da equipe
4 git pull origin dev
5 # Crie a sua branch temporária usando o prefixo adequado
6 git checkout -b feature/tela-cadastro-paciente
Copilot Médico Use o prefixo 'feature/' para telas, 'fix/' para bugs e 'test/' para testes.
Mensagens de Commits Slide 5/7
❌ Commits Inadequados
git commit -m "alterei o banco"

Mensagens genéricas não informam o escopo da alteração e tornam o histórico confuso para quem vai revisar.

✅ Commits Padronizados
git commit -m "feat(db): cadastrar CPF e idade de pacientes"
git commit -m "fix(auth): corrigir expiracao do token JWT"

Usamos Conventional Commits. Prefixo claro + escopo + mensagem simples em português.

Copilot Médico Principais tipos: feat (atualização), fix (bug), test (testes)
GitHub e Revisão Slide 6/7

Rótulos (Labels) para Pull Requests no GitHub

🔴

bug

Para alterações que corrigem falhas de execução no sistema.

🟢

atualização

Para novas features, adição de componentes ou novos layouts.

🔵

testes

Para PRs contendo testes unitários e de integração.

Copilot Médico Utilize labels para tornar a fila de PRs do GitHub legível e organizada!
